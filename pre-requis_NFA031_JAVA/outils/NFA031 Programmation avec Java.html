Compilateur

Nous vous conseillons d'utiliser le compilateur JDK distribué par la société Oracle. Il est gratuit. Utilisez le JDK (JAVA Development Kit) en édition standard (SE). Attention, le runtime (JRE--JAVA Runtime Environment) ne suffit pas.

Vous pouvez le trouver à l'adresse: http://www.oracle.com/technetwork/java/javase/downloads/index.html .

Environnement de développement pour NFA031

Pour le cours NFA031, nous vous conseillons d'utiliser l'environnement de développement drjava qui est très simple à utiliser et qui ne nécessite même pas d'installation. En revanche, il ne peut fonctionner que si le compilateur jdk est installé.

Dr java peut être téléchargé sur le site http://www.drjava.org

Un tutoriel vidéo vous explique comment utiliser drjava (après avoir installé le compilateur).

Autre solution: un éditeur

Pour écrire vos programmes Java, vous pouvez éventuellement un éditeur de texte. Si vous n'en avez pas, nous vous conseillons l'éditeur JEdit que vous pouvez télécharger à l'adresse suivante (choisir la version stable correspondant à votre système d'exploitation)
http://www.jedit.org/index.php?page=download.

Environnement de développement pour NFA032

Pour le cours NFA032, nous vous conseillons d'utiliser l'environnement de développement eclipse qui est un environnement puissant. Vous pouvez également utiliser drjava qui est plus simple à prendre en main.

Vous pouvez télécharger la version Eclipse IDE for Java EE Developers sur le site http://eclipse.org/downloads/

Utilisation du compilateur dans une fenêtre de commande

En windows, pour pouvoir utiliser le compilateur dans une fenêtre de commande, il est utile de mettre à jour la variable d'environnement PATH .

Pour Windows suivre les instructions données dans le cours à distance :
L'installation du JDK pas à pas
Mise à jour du chemin d'exécution, pas à pas

Pour windows 98 et Millenium :
Modifier (ou ajouter) les variables d'environement dans le fichier autoexec.bat (sur C:\).
PATH=...;C:\jdk1.5.0\bin
JAVA_HOME=C:\jdk1.5.0

Pour windows Vista:
A priori, la mise à jour de la variable d'environnement se fait automatiquement. Si ce n'est pas le cas, la procédure de mise à jour peut se faire manuellement de façon similaire à celle de windows XP.
